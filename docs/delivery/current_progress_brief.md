# 当前进展说明

更新时间：2026-06-10

## 一句话进展

项目主线已调整并落实为“基于 SuperMap GIS 三维底座的无人机视觉自主导航仿真系统”。当前已完成珞珈山 scene/map/data 服务发布与前后端接入，v0.5a 已实现 OpenCV ORB 半真实视觉定位原型，并能驱动后端权威导航时间线、三维连续飞行、遥测状态、事件流和导航质量报告。

## 当前已完成

- SuperMap 底座：iDesktopX、iServer、iClient3D 已完成安装与链路验证。
- 项目服务：`3D-luojia_mountain_demo`、`map-luojia_mountain_demo`、`data-luojia_mountain_demo` 已发布并通过脚本门禁。
- 数据场景：珞珈山正射影像、DEM、地形点和建筑面已导入 SuperMap 工作空间并发布服务。
- 后端主线：`VisualNavigationService` 已成为 UAV 导航状态权威源，输出 `reference_position`、`visual_position`、`fused_position`、遥测、事件和导航模式。
- 视觉定位：v0.5a `opencv_orb` provider 已完成，支持关键点提取、匹配、RANSAC、位置估计、置信度和误差半径输出。
- 导航融合：ORB 视觉观测可进入导航时间线，当前 27 帧视觉观测均来自 `opencv_orb`。
- 飞行平滑：后端已完成距离参数化、轨迹圆角平滑、视觉修正渐入渐出，最大单步航向变化约 `14.48°`。
- 报告输出：任务报告新增“视觉导航质量”，可展示平均置信度、平均误差半径、融合轨迹偏差、终点误差和回退帧数。
- 一键验收：`scripts/check_v05_navigation_gate.ps1` 已通过，覆盖后端测试、前端构建、ORB 证据生成、导航/报告接口、DOM 和截图证据。

## 对外汇报推荐说法

> 当前系统已形成 SuperMap 三维场景、半真实 UAV 影像帧、ORB 视觉定位、后端融合导航状态、三维连续飞行、风险/重规划支撑和质量报告输出的演示闭环。该闭环属于软件仿真验证，不接真实飞控；UAV 图像帧为基于正射影像、航线、DEM、建筑和相机参数生成的半真实演示帧，不等同于真实飞行相机数据。

## 严格避免的说法

- 不说“已完成真实飞行视觉自主导航”。
- 不说“系统可以控制真实无人机”。
- 不说“ORB 结果来自真实飞行相机数据”。
- 不说“三维场景已经达到真实倾斜摄影或精细建模效果”。
- 不把航线动画包装成视觉自主导航，必须强调 `fused_position` 来自后端融合时间线。
