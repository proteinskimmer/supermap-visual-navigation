# 答辩讲稿与问答口径

## 1. 开场讲稿

各位老师好，我们的项目是《基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统》。

本项目定位不是直接控制真实无人机，而是面向低空巡检、应急推演和训练验证场景，构建一个软件化的视觉自主导航仿真平台。系统基于 SuperMap GIS 提供的三维空间环境，将 UAV 影像帧与地理场景候选区域进行匹配，形成视觉定位状态，并结合参考航线、风险校验和动态重规划完成任务推演。

## 2. 项目背景讲稿

低空任务和普通路径规划不同，它受到地形、建筑、风险区、禁飞区、通信塔等空间要素影响。单纯在二维地图上画线，很难解释这条航线为什么安全，遇到临时风险时也难以直观展示如何调整。

因此我们选择以 SuperMap GIS 作为底座，用 iDesktopX 整理任务区域数据和三维场景，用 iServer 发布地图、三维和数据服务，再通过 Web 前端进行三维展示和交互。

目前项目自建 `map-low_altitude_demo` 和 `data-low_altitude_demo` 已经完成发布与 REST 验收，前端能够读取并展示 SuperMap 服务状态。三维方面，官方 `3D-CBD` 服务已完成 iClient3D 加载和业务图形叠加验证，项目自建 `3D-low_altitude_demo` 正在进行三维场景制作与发布门禁。

## 3. 系统架构讲稿

系统分为四层。

第一层是 GIS 数据层，包括遥感影像、DEM、任务区域、风险区、障碍物和视觉候选瓦片。

第二层是 SuperMap GIS 服务层。iDesktopX 负责数据生产和三维场景制作，iServer 负责发布三维服务、地图服务和数据服务。

第三层是后端算法服务层，使用 FastAPI 管理任务、参考航线、风险校验、仿真事件和视觉匹配结果。

第四层是前端交互层，使用 Vue 和 SuperMap iClient3D for WebGL 展示三维任务场景、航线、风险区、动态重规划和视觉匹配结果。

## 4. 功能讲稿

系统首先加载任务区域、风险图层和 UAV 影像帧。航线规划模块会生成三条候选参考航线，分别对应最短、最安全和综合最优策略。

每条航线都会经过风险校验，系统会给出评分、风险等级、风险航段和高程剖面。这样答辩时我们不仅能展示“画出了一条线”，还能解释为什么这条航线更合理。

在仿真过程中，系统可以加入临时风险区，例如山火、塌方或临时禁飞区。系统检测到当前航线受影响后，会触发局部重规划，保留已飞航段，并从当前位置生成新的接续航线。

视觉模块当前采用预计算演示框架。输入无人机视角图片后，系统返回候选地理瓦片、置信度、匹配点、内点比例和解释原因。高置信结果在视觉自主模式下用于修正 UAV 导航状态；低置信结果会降级为辅助导航或人工复核提示。后续如果接入 DINOv2、LoFTR、LightGlue 或 OpenCV RANSAC，可以替换后端 provider，不改变前端展示接口。

## 5. SuperMap 使用口径

本项目中 SuperMap 不是普通底图，而是 GIS 底座。

iDesktopX 用于整理影像、DEM、矢量数据和三维场景。

iServer 用于发布三维服务、地图服务和数据服务。

iClient3D for WebGL 用于在前端加载三维场景，并叠加航线、风险区、重规划结果和视觉候选区域。

当前可以确认的是：项目自建 map/data 服务已经进入 SuperMap 服务体系；三维链路已经通过官方 3D 样例验证；项目自建 3D 服务仍在制作和发布中，答辩时我们会按这个状态如实说明。

## 6. 边界说明

本项目聚焦软件仿真中的视觉自主导航状态更新和任务推演，不接入真实无人机飞控，不生成真实飞行控制指令，也不涉及真实武器控制或打击任务。

视觉匹配模块在比赛版本中是软件仿真的导航状态更新链路，不作为真实飞控定位闭环。系统保留辅助导航模式，用于低置信匹配、人工复核和现场演示降级。

## 7. 结尾讲稿

总结来说，我们的系统把 SuperMap GIS 三维底座、UAV 影像匹配、视觉定位状态、参考航线、风险校验和动态重规划整合在一个可演示、可解释、可扩展的平台中。当前重点是保证比赛演示稳定，后续可以继续接入真实 GIS 数据、真实视觉模型和更复杂的低空任务场景。

从当前进度看，业务闭环、map/data 服务发布、前端 SuperMap 状态展示和三维样例链路已经具备验收证据；接下来重点补齐项目自建 3D 场景发布和最终 PPT、视频材料。

## 8. 常见问题答辩口径

### Q1：你们是否控制真实无人机？

答：不控制。当前系统是软件仿真和任务辅助规划平台，不接真实飞控，不生成真实飞行控制指令。真实飞控对接涉及硬件安全、通信延迟、失控保护和飞行审批，不作为本次比赛范围。

### Q2：SuperMap 在项目里具体发挥什么作用？

答：SuperMap 提供 GIS 数据生产、服务发布和三维空间展示能力。iDesktopX 负责数据整理和三维场景制作，iServer 负责发布三维、地图和数据服务，iClient3D for WebGL 负责前端三维可视化。

### Q3：视觉匹配是真实算法还是演示结果？

答：当前阶段采用预计算演示结果，重点先完成稳定的接口、视觉定位状态和导航决策展示框架。系统已经保留 provider 替换点，后续可以接入 DINOv2 做粗检索，LoFTR 或 LightGlue 做局部匹配，OpenCV RANSAC 做几何验证。

### Q4：航线规划算法是什么？

答：当前原型主算法是 A* 栅格路径搜索。我们把 GIS 风险区、障碍物、安全距离和航程作为代价或约束，生成最短、最安全、综合最优三种航线。

### Q5：如何证明航线更安全？

答：系统不仅展示航线，还输出风险评分、风险等级、风险航段、风险原因和高程剖面。这样可以解释航线为什么绕开某些区域，为什么某些航段存在风险。

### Q6：动态重规划如何实现？

答：仿真过程中加入临时风险区后，系统检查当前航线剩余段是否受影响。如果受影响，就以当前位置为新起点，目标点不变，重新规划接续航线，并把原风险航段和新航线同时展示。

### Q7：如果 SuperMap 服务现场加载失败怎么办？

答：前端保留了 `MockMissionMap.vue` 备用演示图。真实三维接入集中在 `SuperMapScene.vue`，如果现场服务异常，可以切回 mock 视图保证演示主流程不断。

### Q8：项目自建 3D 场景现在完成了吗？

答：项目自建 `3D-low_altitude_demo` 还在制作和发布门禁中，不能说已经完成。当前已完成的是项目自建 map/data 服务发布与 REST 验收，以及官方 `3D-CBD` 三维服务到前端 iClient3D 的加载验证。下一步是在 `low_altitude_demo.smwu` 中保存真实三维场景并发布项目自建 3D 服务。

### Q9：项目创新点是什么？

答：第一，以 UAV 影像匹配驱动软件仿真中的视觉自主导航状态更新，而不是只做航线展示；第二，基于 SuperMap 三维 GIS 底座呈现低空任务空间约束；第三，参考航线规划和风险校验结果可解释；第四，支持临时风险触发动态重规划；第五，视觉匹配模块采用可替换 provider，兼顾当前稳定演示和后续算法扩展。
