# 演示视频分镜脚本

建议视频时长控制在 6 到 8 分钟。录制时优先保证主流程稳定，不临时切换数据。

## 录制前准备

- 后端已启动。
- 前端已启动。
- 如果 SuperMap 已接入，iServer 服务已启动。
- 如果 SuperMap 未接入，使用 mock 态势图录制。
- 浏览器缩放建议 100%。
- 关闭无关窗口和通知。
- 使用同一套 demo 数据。

## 分镜脚本

| 时间 | 画面 | 操作 | 旁白 |
|---|---|---|---|
| 0:00-0:20 | 系统首页/工作台全景 | 打开系统 | 这是基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统，面向低空巡检和应急推演场景。 |
| 0:20-0:50 | 任务和图层面板 | 展示任务、图层、任务区域 | 系统首先加载任务区域、地形影像、风险区和障碍物等 GIS 数据。 |
| 0:50-1:30 | 三维/态势图区 | 展示风险区、障碍物、起点和目标点 | 这些空间要素会参与后续航线规划和风险校验。 |
| 1:30-2:10 | 点击规划 | 点击“规划”按钮，显示三条航线 | 系统基于起点、目标点和 GIS 约束，自动生成最短、最安全和综合最优三种候选航线。 |
| 2:10-2:50 | 候选航线列表 | 依次点击三条航线 | 每条航线都有距离、评分、风险等级和规划策略说明。 |
| 2:50-3:30 | 风险校验面板 | 展示风险评分、高程剖面、风险原因 | 系统不仅给出航线，还能解释航线风险来源，包括风险区、障碍物和航程约束。 |
| 3:30-4:10 | 仿真控制 | 点击“仿真”和“播放” | 选择综合最优航线后，系统开始三维仿真，底部事件日志同步记录任务过程。 |
| 4:10-5:00 | 临时风险区 | 点击“重规划” | 在飞行过程中加入临时风险区，系统检测当前航线受影响，触发动态重规划。 |
| 5:00-5:40 | 新旧航线对比 | 展示原航线风险段和新航线 | 原航线风险航段被高亮，新航线从当前位置接续到目标点。 |
| 5:40-6:30 | 视觉匹配 | 切换视觉样例，展示候选区 | 视觉模块输入无人机视角图像，返回候选地理瓦片、置信度、匹配点和解释原因。 |
| 6:30-7:10 | 任务报告 | 点击“报告” | 系统生成任务摘要，汇总推荐航线、风险评分、事件和视觉匹配结果。 |
| 7:10-7:40 | 总结页/工作台全景 | 保持画面 | 本项目聚焦软件仿真验证，不接真实飞控。后续可接入真实 SuperMap 服务、真实视觉模型和更多低空场景。 |

## 录制注意事项

- 如果真实 SuperMap 场景加载慢，先录制 mock 版本，后续补三维场景截图。
- 不要在视频中展示无关路径、账号、授权信息。
- 旁白中不要说系统可以直接控制无人机。
- 视觉匹配必须说“定位辅助”和“预计算演示框架”。
- 如果页面出现加载失败，暂停录制，修复后重新录。
