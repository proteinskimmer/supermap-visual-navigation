# 演示视频分镜脚本

建议视频时长控制在 6 到 8 分钟。录制主线围绕“UAV 影像帧 -> ORB 视觉定位 -> 后端融合导航状态 -> 三维连续飞行 -> 报告输出”展开。

## 录制前准备

- 后端已启动。
- 前端已启动。
- iServer 已启动，珞珈山 `scene/map/data` 服务可访问。
- v0.5 一键门禁已通过。
- 浏览器缩放建议 100%。
- 关闭无关窗口和通知。
- 使用同一套珞珈山 demo 数据。

## 分镜脚本

| 时间 | 画面 | 操作 | 旁白 |
|---|---|---|---|
| 0:00-0:20 | 指挥舱全景 | 打开系统 | 这是基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统。 |
| 0:20-0:50 | SuperMap 场景状态 | 展示任务、图层、scene/map/data 状态 | 系统加载珞珈山三维场景、正射影像、DEM、建筑和任务图层。 |
| 0:50-1:30 | 参考航线与融合轨迹 | 展示三维场景中的起终点、参考航线、风险区 | 参考航线为无人机提供任务约束，后续真实展示以融合导航位置为准。 |
| 1:30-2:20 | 视觉自主模式 | 点击推演/播放 | UAV 影像帧进入后端，系统调用 ORB provider 进行视觉地理重定位。 |
| 2:20-3:10 | 视觉影像与匹配状态 | 切换 UAV 帧或展示视觉面板 | 系统输出视觉位置、置信度、误差半径和匹配证据，低置信时可进入辅助或复核状态。 |
| 3:10-4:00 | 三维连续飞行 | 播放一段飞行 | 三维无人机按后端 `fused_position` 连续飞行，遥测、航向、偏差和事件流同步更新。 |
| 4:00-4:50 | 参考/融合轨迹对比 | 展示两条轨迹 | 参考轨迹来自航线规划，融合轨迹来自视觉定位和后端状态融合，用于解释实际导航偏差。 |
| 4:50-5:30 | 风险与重规划 | 触发或展示风险事件 | 临时风险出现时，系统基于当前融合位置接续重规划，而不是从固定航点重新开始。 |
| 5:30-6:30 | 任务报告 | 打开报告页 | 报告汇总航线、风险、高程剖面和视觉导航质量，包括平均置信度、误差半径和轨迹偏差。 |
| 6:30-7:10 | 当前验收数据 | 展示 PPT 或报告摘要 | 当前 v0.5a 已通过一键门禁，6/6 半真实 UAV 帧定位成功，27 帧视觉观测来自 opencv_orb。 |
| 7:10-7:40 | 总结画面 | 回到指挥舱全景 | 本项目聚焦软件仿真验证，不接真实飞控；后续将补真实 UAV 相机数据和更强视觉匹配模型。 |

## 录制注意事项

- 不说系统可以控制真实无人机。
- 不说当前已经完成真实飞行视觉自主导航。
- 必须说明 UAV 图像帧是半真实演示帧，不是真实飞行相机数据。
- 必须说明三维场景是演示级 GIS 场景，不是精细倾斜摄影建模。
- 如果 iServer 或 WebGL 出现异常，可使用本地 Luojia fallback 场景，但旁白要说明是演示兜底。
