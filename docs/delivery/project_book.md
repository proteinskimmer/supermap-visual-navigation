# 项目书

## 一、项目名称

基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统

## 二、项目定位

本项目面向低空巡检、应急推演和训练验证场景，建设一个基于 SuperMap GIS 三维底座的无人机视觉自主导航仿真系统。系统以珞珈山三维 GIS 场景为任务空间，以 UAV 视角影像帧为视觉输入，通过视觉地理重定位获得位置估计，再由后端融合生成权威导航状态，驱动三维无人机、遥测面板、事件流、风险判断、重规划和任务报告。

项目当前阶段定位为软件仿真验证平台，不接入真实无人机飞控，不生成真实飞行控制指令。当前 UAV 图像帧为基于正射影像、航线、DEM、建筑和相机参数生成的半真实演示帧，不等同于真实飞行相机数据。系统成果用于比赛演示、算法接口验证和任务推演，不声明已完成真实飞行环境下的自主导航闭环。

## 三、建设背景

低空任务不只是“画一条航线”。在巡检、应急和训练推演中，无人机需要在复杂地形、建筑、风险区和临时事件环境下持续判断自身位置与任务目标的关系。传统二维地图或固定航线动画可以展示路线，但难以解释无人机如何根据视觉图像更新位置，也难以量化定位置信度、误差半径和轨迹偏差。

SuperMap GIS 能够提供影像、DEM、建筑、风险区和三维场景服务，为视觉导航提供空间参照。视觉定位算法则可以把 UAV 视角图像与正射影像或候选合成视图进行匹配，形成视觉位置估计。将二者结合后，系统可以进一步构建参考位置、视觉位置和融合位置的导航状态链路，使任务推演从“航线展示”升级为“视觉定位驱动的导航状态仿真”。

## 四、建设目标

本项目的总体目标是形成一个可演示、可解释、可复验的低空视觉自主导航仿真平台。

具体目标包括：

- 构建基于 SuperMap 的珞珈山 scene/map/data 服务，支撑正射影像、DEM、建筑、航线、风险区和任务要素展示。
- 建立后端权威导航状态机，输出参考位置、视觉位置、融合位置、遥测数据、导航模式和事件流。
- 实现 OpenCV ORB 视觉定位最小原型，对半真实 UAV 影像帧与候选合成视图进行真实特征匹配。
- 将视觉定位结果写入导航时间线，驱动三维无人机按 `fused_position` 连续飞行。
- 保留航线规划、风险校验和动态重规划能力，作为视觉自主导航任务的安全支撑服务。
- 输出导航质量报告，量化展示置信度、误差半径、融合轨迹偏差、终点误差和回退情况。
- 完成 PPT、答辩讲稿、演示视频脚本、截图证据和提交包模板。

## 五、总体方案

系统采用“SuperMap GIS 底座 + FastAPI 后端导航状态机 + Vue 前端指挥舱 + 可替换视觉匹配 provider”的总体架构。

数据层提供珞珈山正射影像、DEM、地形点、建筑面、任务区域、风险区、参考航线、半真实 UAV 图像帧和候选合成视图。GIS 服务层由 SuperMap iDesktopX 与 iServer 承担，负责构建工作空间、发布 `3D-luojia_mountain_demo`、`map-luojia_mountain_demo` 和 `data-luojia_mountain_demo` 服务。前端通过 iClient3D 加载三维场景，并叠加参考轨迹、融合轨迹、风险区、视觉候选区和 UAV 当前状态。

后端采用 Python 与 FastAPI 实现，核心服务包括任务管理、航线规划、风险校验、视觉定位、导航融合、仿真事件和报告输出。其中 `VisualNavigationService` 是 UAV 导航状态的权威源，前端只消费后端时间线，不自行推演 UAV 状态。

视觉定位采用 provider 设计。当前 v0.5a 已接入 `opencv_orb` provider，支持 ORB 关键点提取、特征匹配、RANSAC 几何验证、位置估计、置信度和误差半径输出。后续可扩展 SIFT、LoFTR、LightGlue 或外部深度匹配器。

## 六、核心建设内容

### 1. SuperMap 三维 GIS 底座

系统已完成珞珈山 SuperMap 工作空间和服务发布，当前 scene/map/data 服务通过脚本门禁验收。三维场景用于承载正射影像、DEM、建筑、参考航线、融合轨迹和风险要素，是视觉导航状态展示的空间基础。

### 2. 视觉定位与 ORB 匹配

系统生成半真实 UAV 图像帧，并根据当前航线、相机参数、视场角、光照和镜头畸变构造演示输入。`opencv_orb` provider 对 UAV 帧和候选合成视图进行特征匹配，通过 RANSAC 输出定位估计、匹配点数、内点比例、置信度和误差半径。

当前验收结果显示，`6/6` 个半真实 UAV 帧可完成 ORB 定位，导航时间线中 `27` 帧视觉观测均来自 `opencv_orb`。

### 3. 后端融合导航状态

后端维护统一导航时间线，核心字段包括：

- `reference_position`：参考航线位置。
- `visual_position`：视觉定位位置。
- `fused_position`：融合后的权威导航位置。
- `navigation_mode`：autonomous、assisted、review。
- `telemetry`：速度、航向、姿态、电量和信号。
- `event_stream`：视觉定位、风险、复核、重规划和报告事件。

前端三维 UAV、遥测面板、视觉状态、事件流和报告均消费同一条后端时间线，保证演示状态一致。

### 4. 三维连续飞行与指挥舱

前端指挥舱在 SuperMap/iClient3D 场景中展示参考轨迹和融合轨迹。系统已完成轨迹距离参数化、航线圆角平滑、视觉修正渐入渐出和播放插值，修正后最大单步航向变化约 `14.48°`，避免无人机出现不自然硬转向。

### 5. 规划、风险与动态重规划

航线规划、风险校验和动态重规划作为视觉自主导航任务的支撑能力保留。系统可生成多策略参考航线，对风险区、障碍物、航程和高程进行校验，并在临时风险出现时基于当前 `fused_position` 接续重规划。

### 6. 导航质量报告

任务报告新增视觉导航质量板块，统计 provider、视觉观测数量、平均置信度、误差半径、融合轨迹偏差、终点误差、最大速度、航向平滑性和回退帧数。当前质量等级为 `demo_verified`。

## 九、项目创新与特色

- 以 SuperMap 三维 GIS 作为视觉导航的空间参照，而非普通底图。
- 将视觉匹配结果写入后端导航状态机，驱动 `fused_position` 连续飞行。
- 使用 ORB 特征匹配和 RANSAC 构建可解释的最小视觉定位原型。
- 将定位质量量化为置信度、误差半径、轨迹偏差和回退帧。
- 航线规划、风险校验和重规划围绕当前融合位置提供安全支撑。
- provider 架构可平滑扩展到 SIFT、LoFTR、LightGlue 或其他深度匹配模型。

## 十、风险与应对

真实 UAV 图像暂未入库。应对方式是明确当前使用半真实演示帧，并记录相机参数、光照、畸变和位姿假设，后续可替换真实飞行数据。

视觉匹配算法对纹理、光照和尺度变化敏感。应对方式是保留 provider 降级机制，并在报告中展示回退帧和质量等级，避免把不稳定结果包装成真实闭环。

三维场景当前为演示级 GIS 场景。应对方式是在答辩中明确它用于空间关系展示，不宣称精细倾斜摄影效果。

真实飞控涉及硬件安全、通信链路和飞行审批。应对方式是将当前边界限定为软件仿真验证，真实飞控仅作为未来扩展方向。

## 十一、结论

本项目已从单纯航线规划展示升级为视觉定位驱动的自主导航仿真系统。当前 SuperMap 珞珈山三维底座、ORB 视觉定位、后端融合导航状态、前端三维指挥舱、风险重规划支撑和导航质量报告已经形成可演示闭环。项目具备比赛答辩展示基础，也为后续接入真实 UAV 图像、深度匹配模型和真实任务场景留下了清晰扩展路径。
