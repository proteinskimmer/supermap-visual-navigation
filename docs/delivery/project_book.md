# 项目书

## 一、项目名称

基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统

## 二、项目定位

本项目面向低空巡检、应急推演和训练验证等场景，建设一个基于 SuperMap GIS 三维环境的低空任务智能规划与仿真验证平台。系统以空间数据为基础，以航线规划、风险校验、动态重规划和视觉定位辅助为核心能力，形成从任务区域构建、候选航线生成、风险分析、仿真推演到报告输出的完整软件闭环。

项目当前阶段定位为软件系统开发与仿真验证，不接入真实无人机飞控，不生成真实飞行控制指令。视觉相关能力用于图像与地理候选区域的匹配展示和算法接入验证，不声明已形成真实飞行环境下的实时定位闭环。

## 三、建设背景

随着低空经济、无人机巡检和应急处置需求的发展，低空任务不再只是简单的航点连接问题，而是需要在复杂地理环境中综合考虑地形起伏、建筑障碍、限制区域、风险事件和任务目标。传统二维地图虽然能够展示平面位置，但对高度、障碍、风险区与航线之间的空间关系表达不足，难以支撑任务过程的解释、校验和复盘。

同时，低空任务现场可能出现临时风险，例如山火扩散、道路塌方、临时管制区或障碍物变化。人工重新规划航线耗时较长，且难以在短时间内解释新旧航线的安全性差异。视觉图像也常常需要与遥感影像或地理底图进行对应，辅助判断当前位置或任务区域状态。

因此，本项目提出以 SuperMap GIS 为三维空间底座，将 GIS 数据生产、服务发布、智能规划、风险校验、视觉匹配和仿真展示进行集成，构建一个可演示、可解释、可扩展的低空任务辅助规划平台。

## 四、建设目标

本项目的总体目标是完成一个面向低空任务的智能规划与三维仿真原型系统，并具备比赛答辩和演示所需的完整材料支撑。

具体目标包括：

- 构建基于 SuperMap 的任务区域三维 GIS 环境，支撑影像、地形、矢量地物、风险区和航线的综合展示。
- 实现基于起点、目标点、任务区域和 GIS 约束的多策略航线规划，输出最短航线、最安全航线和综合最优航线。
- 实现航线风险校验，对风险区穿越、障碍接近、边界越界、高程变化和航程约束进行分析，并输出评分和解释。
- 实现动态重规划仿真，在新增临时风险区后自动检测受影响航段，并生成接续新航线。
- 建立视觉定位辅助框架，将无人机视角图像与遥感瓦片进行候选区域匹配，并在三维场景中展示候选位置和置信度。
- 形成项目文档、部署说明、数据说明、PPT、演示视频脚本和提交包模板，支撑项目验收与比赛提交。

## 五、总体方案

系统采用“SuperMap GIS 底座 + Python 后端服务 + Vue 前端工作台 + 可替换算法模块”的总体架构。

数据层负责提供遥感影像、DEM 高程、道路水系、建筑物、任务边界、风险区、障碍物、任务点和视觉瓦片索引等基础数据。GIS 服务层由 SuperMap iDesktopX 和 iServer 承担，负责数据整理、三维场景制作、地图服务发布、三维服务发布和空间数据服务输出。

后端服务层采用 Python 与 FastAPI 实现，负责任务管理、航线规划、风险校验、视觉匹配、仿真事件生成和报告摘要输出。前端交互层采用 Vue，后续接入 SuperMap iClient3D for Cesium，用于三维场景加载、图层控制、航线交互、风险展示、仿真播放和报告查看。

算法层采用可替换设计。当前原型使用 A* 栅格规划实现航线生成，使用规则评分完成风险校验，使用预计算结果完成视觉匹配演示。后续可替换或增强为 RRT*、DINOv2、LoFTR、LightGlue、OpenCV RANSAC 等算法能力。

## 六、核心建设内容

### 1. SuperMap 三维 GIS 环境

使用 SuperMap iDesktopX 对任务区域影像、DEM、矢量数据和三维场景进行整理与制作，并通过 SuperMap iServer 发布地图服务、三维服务和数据服务。前端通过 SuperMap iClient3D 加载场景，在三维环境中叠加风险区、障碍物、航线、候选视觉匹配区域和仿真轨迹。

该部分的价值在于将低空任务从普通平面展示提升到真实地理空间中的三维推演，使航线与地形、障碍和风险范围之间的关系可以被直观解释。

### 2. 智能航线规划

系统根据起点、目标点、任务区域、风险区、障碍物和安全参数自动生成候选航线。当前原型提供最短航线、最安全航线和综合最优航线三种规划模式，分别服务于效率优先、安全优先和平衡优化的任务需求。

规划结果不仅展示为线状轨迹，还同步输出航程、风险评分、风险等级和推荐原因，方便用户对不同方案进行比较和选择。

### 3. 航线风险校验

系统对人工航线和自动航线进行统一风险校验，分析航线是否穿越风险区、是否接近障碍物、是否超出任务边界、是否存在高程突变、是否经过不适宜区域以及是否超出最大航程。

风险校验结果以评分、等级、风险航段、原因说明和高程剖面形式输出，使系统具备“给出结论”和“解释结论”的双重能力。

### 4. 动态重规划仿真

在仿真播放过程中，系统支持加入临时风险区。当新增风险区与当前航线发生冲突时，系统自动识别受影响航段，触发局部重规划，并将原航线风险段、新航线和事件日志同步展示。

该功能用于模拟低空任务中的突发情况处置过程，增强系统的任务故事性、演示性和复盘能力。

### 5. 视觉定位辅助

视觉模块用于展示无人机视角图像与遥感瓦片之间的候选匹配关系。系统可输出候选瓦片、置信度、匹配点数量、内点比例和解释信息，并在三维 GIS 场景中高亮候选位置。

当前阶段采用预计算结果保证比赛演示稳定性，同时保留真实算法接入接口。后续可引入 DINOv2 进行粗检索，使用 LoFTR 或 LightGlue 进行局部特征匹配，并结合 RANSAC 完成几何一致性验证。

### 6. 三维仿真与报告输出

前端工作台提供任务区展示、图层切换、航线规划、仿真播放、事件日志、视觉匹配结果和任务报告查看。任务报告可汇总任务区域、候选航线、风险校验结果、重规划事件和视觉匹配结果，为答辩演示和项目验收提供结构化输出。

## 七、技术路线

本项目采用分阶段、可演示优先的技术路线。

第一阶段完成 SuperMap 底座跑通，包括 iDesktopX 数据整理、iServer 服务发布和前端场景加载。第二阶段完成 Vue 与 FastAPI 工程骨架，形成任务管理、图层管理和基础展示能力。第三阶段实现航线规划、风险校验和高程剖面，形成可解释的路线比较能力。第四阶段完成视觉匹配框架，先以预计算结果稳定演示，再预留真实模型接入。第五阶段实现动态重规划和完整仿真流程。第六阶段整理 PPT、视频、截图、部署说明和提交包。

在工程实现上，系统坚持接口稳定、数据可替换和演示可兜底的原则。即使真实 SuperMap 服务或视觉模型在现场环境中不可用，系统仍可通过 Mock 数据完整展示业务流程；当真实服务接入后，也可在不大幅改变前端和后端业务接口的情况下替换数据来源。

## 八、实施计划

### 阶段一：SuperMap 底座建设

完成 SuperMap iDesktopX、iServer 和 iClient3D 相关环境准备，制作任务区域数据，发布地图服务和三维服务，实现前端三维场景加载。

交付成果包括 SuperMap 服务截图、三维场景截图、服务地址登记表和接入说明。

### 阶段二：平台基础功能开发

完成后端 FastAPI 接口和前端 Vue 工作台，支持任务加载、图层展示、航线显示、仿真控制和基础报告查看。

交付成果包括可运行的前后端工程、接口说明和基础演示页面。

### 阶段三：航线规划与风险校验

实现多策略航线规划、风险区校验、障碍接近判断、高程剖面和航线评分，支持候选航线对比展示。

交付成果包括航线规划演示、风险评分面板、高程剖面截图和风险校验报告。

### 阶段四：视觉匹配模块

准备遥感瓦片索引和无人机视角样例，完成视觉匹配接口和候选结果展示，形成可替换的算法接入框架。

交付成果包括视觉匹配页面、候选区域高亮截图和算法接入说明。

### 阶段五：动态重规划与仿真闭环

实现临时风险区添加、受影响航段检测、局部重规划、仿真轨迹播放和事件日志记录。

交付成果包括完整 demo、重规划前后对比图和演示视频脚本。

### 阶段六：比赛材料整理

完善系统设计、部署说明、数据说明、源码结构说明、PPT、答辩讲稿、演示视频和最终提交包。

交付成果包括完整参赛材料和提交检查清单。

## 九、预期成果

项目完成后将形成以下成果：

- 一个可运行的低空任务智能规划与三维仿真原型系统。
- 一套基于 SuperMap 的 GIS 数据生产、服务发布和前端接入流程。
- 一组航线规划、风险校验、动态重规划和视觉匹配相关接口。
- 一套演示数据、截图清单、视频脚本和答辩材料。
- 一份系统设计说明、部署说明、数据说明、源码结构说明和最终提交包模板。

## 十、项目创新与特色

本项目的创新点主要体现在以下方面：

- 以 SuperMap 三维 GIS 为任务空间底座，使航线规划结果能够在真实地理环境中展示、校验和解释。
- 将航线生成、风险评分、风险原因说明和高程剖面结合，提升规划结果的可解释性。
- 通过临时风险区触发动态重规划，模拟真实任务中突发事件处置过程。
- 视觉匹配采用可替换框架，兼顾比赛演示稳定性和后续算法升级空间。
- 采用 Mock 数据与真实服务双路径设计，降低现场演示风险，提高工程落地弹性。

## 十一、风险与应对

SuperMap 真实服务接入可能受安装环境、授权和数据准备影响。应对方式是提前保留 Mock 场景和服务配置模板，确保演示流程不受单一环境限制。

视觉匹配模型对算力和数据质量要求较高。应对方式是在比赛阶段采用预计算结果稳定展示，同时明确真实模型接入方案和接口位置。

真实无人机飞控涉及硬件安全、通信链路、传感器误差和飞行审批。应对方式是将当前项目边界限定为软件仿真和辅助规划，真实飞控仅作为未来扩展方向。

## 十二、结论

本项目围绕低空任务中的空间感知、航线规划、风险解释和仿真验证需求，提出并实现基于 SuperMap GIS 底座的智能规划平台。项目能够把 GIS 数据、三维场景、算法服务和前端交互整合为完整演示闭环，既具备比赛展示价值，也具备向真实数据、真实模型和更多低空任务场景扩展的基础。
