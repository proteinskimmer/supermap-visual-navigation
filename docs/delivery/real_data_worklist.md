# 真实数据工作清单

本清单用于从 mock 数据过渡到真实 SuperMap GIS 数据。真实数据不必一次到位，优先满足可演示、可解释、可发布。

详细下发给组员的数据采集任务书见：

```text
docs/project_management/14_real_data_collection_guide.md
```

组员提交目录建议为：

```text
data_sources/低空巡检示范区_YYYYMMDD/
```

## 1. 任务区域选择标准

优先选择满足以下条件的区域：

- 范围适中，建议 3 km x 3 km 到 10 km x 10 km。
- 有明显地形或地物差异。
- 有道路、水系、建筑或开阔地。
- 可以合理设置风险区和障碍物。
- 数据来源公开、可说明。
- 不涉及敏感区域。

不建议：

- 区域过大，导致数据处理和三维加载过慢。
- 地物过于单一，视觉匹配和风险规划展示效果弱。
- 数据来源无法说明。

## 2. 数据来源建议

| 数据 | 建议来源 | 备注 |
|---|---|---|
| 任务区域边界 | 手工绘制/公开行政边界裁剪 | 可在 iDesktopX 中绘制 |
| 遥感影像 | SuperMap 样例数据/公开影像 | 比赛阶段优先用稳定数据 |
| DEM | SuperMap 样例数据/公开 DEM | 用于地形和高程剖面 |
| 道路 | OSM/公开矢量/手工绘制 | 用于场景表达 |
| 水系 | OSM/公开矢量/手工绘制 | 用于视觉和场景表达 |
| 建筑 | OSM/公开建筑轮廓/样例数据 | 用于障碍解释 |
| 风险区 | 手工绘制 | 最适合演示控制 |
| 障碍物 | 手工标注 | 通信塔、高层建筑、危险地形 |
| 视觉样例图 | 真实航拍/模拟截图/公开图片 | 必须能和任务区解释对应 |

## 3. 数据交付格式

GIS 负责人至少交付：

| 交付物 | 格式 | 用途 |
|---|---|---|
| 工作空间 | SuperMap 工作空间 | iServer 发布 |
| 任务区域边界 | SuperMap 数据集/GeoJSON | 规划范围 |
| 风险区 | SuperMap 数据集/GeoJSON | 风险校验和展示 |
| 障碍物 | SuperMap 数据集/GeoJSON | 风险解释 |
| 服务地址表 | Markdown/JSON | 前后端联调 |
| 场景截图 | PNG | PPT/视频 |

## 4. 图层字段检查

### risk_zone

- [ ] `id`
- [ ] `name`
- [ ] `type`
- [ ] `level`
- [ ] `buffer_m`
- [ ] `active`

### obstacle

- [ ] `id`
- [ ] `name`
- [ ] `type`
- [ ] `height_m`
- [ ] `buffer_m`

## 5. 坐标检查

- [ ] 所有图层坐标系统一。
- [ ] 前端使用经纬度顺序 `[lon, lat, height]`。
- [ ] 航线和风险区叠加无明显偏移。
- [ ] iServer 服务和导出 GeoJSON 坐标一致。
- [ ] 坐标系信息已写入数据说明。

## 6. 视觉数据检查

如果使用真实或模拟无人机视角图：

- [ ] 每张图有 `image_id`。
- [ ] 每张图有对应的任务时间点。
- [ ] 每张图能解释大致拍摄区域。
- [ ] 每张图有 3 个候选瓦片。
- [ ] 最优候选区域能在地图中高亮。
- [ ] 置信度、匹配点、内点比例和解释原因已填写。

## 7. 从真实数据更新 demo JSON

需要更新：

```text
demo_data/task_demo.json
```

重点字段：

- `task.area`
- `task.start`
- `task.target`
- `risk_zones`
- `obstacles`
- `vision_images`
- `vision_tile_index`
- `vision_matches`
- `layers[].service_url`

如果使用 SuperMap 服务配置，还要更新：

```text
config/supermap_services.local.json
```

## 8. 最小真实数据版本

如果时间紧，最低完成：

- 1 个任务区域。
- 1 个三维服务。
- 1 个数据服务。
- 2 个固定风险区。
- 2 个障碍物。
- 3 条候选航线。
- 1 次动态重规划。
- 1 张视觉样例图或可解释占位图。

## 9. 验收标准

- 真实数据能在 iDesktopX 正确叠加。
- iServer 服务能发布。
- 前端能加载或至少记录服务 URL。
- 航线、风险区、障碍物位置一致。
- 数据来源和用途能在答辩中说明。
