# 分工细化：项目统筹与系统架构

## 1. 角色目标

保证项目方向、范围、进度、接口和演示主线不失控，让各模块能够按统一口径协作并最终形成完整参赛作品。

## 2. 核心职责

- 明确项目边界：软件仿真系统，不接真实飞控。
- 拆分里程碑和每周任务。
- 维护系统架构图、模块关系、接口清单。
- 组织联调、验收和演示彩排。
- 统一答辩口径和比赛材料结构。

## 3. 输入

- 原始项目规划文档。
- 比赛规则和软件要求。
- 各成员每周进度。
- 模块接口变化。
- 演示效果反馈。

## 4. 输出

| 输出物 | 格式 | 验收标准 |
|---|---|---|
| 总体架构图 | Mermaid/图片/PPT | 能讲清 SuperMap、前端、后端、算法关系 |
| 里程碑计划 | Markdown/表格 | 每阶段目标、负责人、交付物明确 |
| 接口清单 | Markdown/OpenAPI | 前后端和算法联调无歧义 |
| 周进度记录 | Markdown | 可看出完成、延期、阻塞 |
| 演示脚本 | Markdown/PPT | 能按 5 到 8 分钟稳定演示 |

## 5. 执行步骤

### 第 1 周

- 确定最终技术路线：SuperMap iDesktopX 2025 + iServer 2025 + iClient3D for WebGL + Vue 3 + FastAPI。
- 整理团队成员和职责表。
- 确定任务区域数据来源。
- 建立任务看板和周会节奏。

### 第 2 周

- 确认前后端项目目录结构。
- 确认接口命名和基础数据结构。
- 推进前端 mock 和后端 API 对齐。
- 检查 SuperMap 服务 URL 是否稳定。

### 第 3 周

- 组织航线规划、风险校验、前端可视化联调。
- 明确风险评分公式的解释口径。
- 开始收集系统截图。

### 第 4 周

- 组织动态重规划演示链路联调。
- 固定演示数据，不再频繁更换任务区域。
- 梳理答辩中的技术亮点。

### 第 5 周

- 组织视觉匹配演示结果接入。
- 检查视觉模块是否影响主流程稳定性。
- 开始录制演示素材。

### 第 6 周

- 冻结功能。
- 完成部署文档、系统说明、PPT、演示视频。
- 做至少 3 次完整彩排。

## 6. 每日检查项

- 是否有成员任务阻塞。
- 是否有接口变更未同步。
- 是否有功能偏离比赛主线。
- 是否有演示链路变得不稳定。

## 7. 风险处理

| 风险 | 处理方式 |
|---|---|
| SuperMap 服务不稳定 | 保留本地服务截图和备用样例服务 |
| 视觉模型效果不稳定 | 使用预计算结果和固定输入图片 |
| 前后端接口拖延 | 前端先用 mock，后端按接口文档补齐 |
| 算法开发超时 | 优先完成 A* 和可解释评分，RRT* 作为扩展 |

## 8. 完成标准

- 每个角色知道本周必须交付什么。
- 项目能按里程碑验收，而不是只停留在概念描述。
- 最终演示可以从打开系统到生成报告完整走通。
