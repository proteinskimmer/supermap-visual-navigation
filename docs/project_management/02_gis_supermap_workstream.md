# 分工细化：SuperMap 与 GIS 数据

## 1. 角色目标

构建项目的 GIS 底座，提供可在 Web 前端加载、可供后端查询和分析的任务区域数据服务。

## 2. 推荐软件

- SuperMap iDesktopX 2025。
- SuperMap iServer 2025。
- SuperMap iClient3D for WebGL 示例和资源。
- QGIS，可选，用于辅助查看公开数据。

## 3. 核心职责

- 获取或制作任务区域基础数据。
- 统一坐标系。
- 整理影像、DEM、矢量图层、风险区、任务点。
- 制作 SuperMap 工作空间和三维场景。
- 发布 iServer 服务。
- 为前端和后端提供服务地址、图层名、字段说明。

## 4. 数据清单

| 数据 | 必需 | 说明 |
|---|---|---|
| 任务区域边界 | 是 | 用于限制航线规划范围 |
| 遥感影像/底图 | 是 | 用于三维场景背景和视觉匹配候选瓦片 |
| DEM 高程 | 是 | 用于地形展示、高程剖面和安全高度校验 |
| 道路、水系、建筑 | 是 | 用于场景展示和风险约束 |
| 风险区/禁入区 | 是 | 可人工绘制，用于航线避让 |
| 临时风险区 | 是 | 用于动态重规划演示 |
| 三维建筑模型 | 可选 | 有则增强演示效果 |
| 无人机视角图像 | 可选 | 视觉匹配演示输入 |

## 5. 执行步骤

### 步骤 1：确定任务区域

- 选择一个范围适中的区域，优先选数据容易获取的区域。
- 范围不要过大，建议 3 km x 3 km 到 10 km x 10 km。
- 输出任务区域边界 GeoJSON 或 SuperMap 数据集。

### 步骤 2：整理数据

- 在 iDesktopX 中导入影像、DEM 和矢量数据。
- 统一坐标系，推荐使用 WGS84 或项目统一坐标。
- 裁剪到任务区域。
- 给风险区、禁入区、障碍物图层添加规范字段。

### 步骤 3：制作三维场景

- 加载影像和地形。
- 添加建筑、道路、水系、风险区。
- 设置合适默认视角。
- 保存工作空间。

### 步骤 4：发布服务

- 在 iServer 发布地图服务。
- 发布三维服务。
- 发布数据服务。
- 如有条件，发布空间分析服务。

### 步骤 5：交付服务信息

向前端和后端提供：

- iServer 根地址。
- 三维场景服务 URL。
- 地图服务 URL。
- 数据服务 URL。
- 图层列表。
- 字段说明。
- 示例查询结果。

## 6. 字段建议

### risk_zone

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 风险区唯一编号 |
| name | string | 风险区名称 |
| type | string | fire/landslide/no_fly/manual |
| level | int | 1 到 5，数值越大风险越高 |
| buffer_m | float | 建议安全缓冲距离 |
| active | bool | 是否启用 |

### obstacle

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 障碍物编号 |
| name | string | 名称 |
| type | string | building/tower/terrain |
| height_m | float | 高度 |
| buffer_m | float | 安全缓冲距离 |

## 7. 对外交付物

| 交付物 | 接收方 |
|---|---|
| iServer 服务地址表 | 前端、后端 |
| 图层字段说明 | 后端、算法 |
| 任务区域边界数据 | 后端、算法 |
| 风险区数据 | 后端、规划算法、前端 |
| 三维场景截图 | 项目统筹、材料负责人 |

## 8. 验收标准

- 前端可以成功加载 SuperMap 三维场景。
- 后端可以通过服务或导出数据读取任务区域、风险区、DEM 信息。
- 风险区和航线能在同一坐标系下正确叠加。
- 至少有一个固定演示区域和一套稳定数据。
