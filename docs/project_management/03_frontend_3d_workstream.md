# 分工细化：前端三维平台

## 1. 角色目标

构建可交互、可演示的三维 Web 平台，把 SuperMap 三维场景、航线规划、风险校验、动态重规划和视觉匹配结果直观展示出来。

## 2. 技术栈

- Vue 3。
- Vite。
- SuperMap iClient3D for WebGL。
- ECharts。
- Axios 或 Fetch。

## 3. 页面结构

| 区域 | 内容 |
|---|---|
| 中央 | SuperMap 三维场景 |
| 左侧 | 任务选择、图层控制、起终点设置 |
| 右侧 | 航线评分、风险告警、视觉匹配结果 |
| 底部 | 仿真时间轴、播放控制、事件日志 |
| 小窗 | 无人机视角图片或匹配点展示 |

## 4. 核心功能拆分

### F1 三维场景加载

- 加载 iServer 三维服务。
- 设置默认视角。
- 显示任务区域边界。
- 提供图层开关。

验收：打开页面后 5 秒内能看到任务区域三维场景。

### F2 任务与航线交互

- 选择示范任务。
- 设置起点、终点、任务点。
- 请求后端生成航线。
- 展示最短、最安全、综合最优三条航线。
- 支持选择一条航线进入仿真。

验收：三条航线颜色区分明确，点击后右侧评分同步变化。

### F3 风险可视化

- 展示风险区、禁入区、障碍物。
- 高亮风险航段。
- 展示风险原因列表。
- 展示航线评分和风险等级。
- 使用 ECharts 展示高程剖面。

验收：用户能看懂为什么某段航线有风险。

### F4 仿真播放

- 播放、暂停、倍速、重置。
- 航迹动态推进。
- 当前飞行点跟随视角。
- 事件日志按时间出现。

验收：可连续播放一次完整任务。

### F5 动态重规划展示

- 支持点击按钮或接口触发临时风险区。
- 原航线风险段变红。
- 新航线接续到目标点。
- 事件日志记录触发原因。

验收：重规划过程在地图上清晰可见。

### F6 视觉匹配展示

- 展示输入图像。
- 展示候选区域。
- 展示置信度、匹配点数量、偏移估计。
- 在三维场景中高亮候选位置。

验收：能够解释视觉匹配只是地理配准辅助，不是飞控输出。

### F7 报告查看

- 展示任务摘要。
- 展示航线评分、风险清单、事件日志。
- 支持导出或生成报告内容。

验收：报告信息能用于比赛材料截图。

## 5. 前端状态模型建议

| 状态 | 说明 |
|---|---|
| currentTask | 当前任务 |
| layers | 图层开关状态 |
| routes | 候选航线列表 |
| selectedRouteId | 当前选择航线 |
| riskAnalysis | 风险校验结果 |
| simulationState | 播放状态、时间、速度 |
| events | 仿真事件日志 |
| visionResult | 视觉匹配结果 |

## 6. 联调依赖

| 依赖 | 提供方 |
|---|---|
| iServer 三维服务 URL | GIS 负责人 |
| 航线规划接口 | 后端、规划算法 |
| 风险校验接口 | 后端、规划算法 |
| 仿真事件接口 | 后端 |
| 视觉匹配结果接口 | 后端、视觉负责人 |

## 7. 验收标准

- 页面布局完整，不依赖开发者控制台操作。
- 三维场景、航线、风险区、视觉候选区域坐标一致。
- 主演示流程可以连续操作。
- 页面显示内容能直接截图用于 PPT。
