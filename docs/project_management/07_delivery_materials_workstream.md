# 分工细化：比赛材料与演示交付

## 1. 角色目标

把系统能力转化为清晰、可信、可答辩的参赛材料，包括系统说明、部署说明、数据说明、PPT、演示视频和截图。

## 2. 核心材料

| 材料 | 说明 |
|---|---|
| 系统介绍文档 | 项目背景、目标、架构、功能、创新点 |
| 部署说明 | 软件安装、服务启动、前后端启动 |
| 数据说明 | 数据来源、图层、坐标系、字段、用途 |
| 源代码目录说明 | 前端、后端、算法、数据、文档结构 |
| PPT | 答辩展示 |
| 演示视频 | 5 到 8 分钟完整流程 |
| 截图素材 | 服务发布、三维场景、航线规划、重规划、视觉匹配 |

## 3. PPT 建议结构

1. 项目名称和一句话定位。
2. 应用场景和问题定义。
3. 系统总体架构。
4. SuperMap 在项目中的作用。
5. 核心功能一：三维 GIS 环境。
6. 核心功能二：航线规划与风险校验。
7. 核心功能三：动态重规划仿真。
8. 核心功能四：视觉定位辅助。
9. 演示流程和效果截图。
10. 创新点、边界和未来扩展。

## 4. 演示视频脚本

| 时间 | 内容 |
|---|---|
| 0:00-0:30 | 项目名称、系统定位 |
| 0:30-1:20 | 加载 SuperMap 三维任务区域 |
| 1:20-2:20 | 展示影像、地形、风险区、障碍物 |
| 2:20-3:20 | 输入起点终点，生成三条候选航线 |
| 3:20-4:10 | 展示风险评分、高程剖面、风险原因 |
| 4:10-5:20 | 播放三维仿真和事件日志 |
| 5:20-6:20 | 加入临时风险区并触发重规划 |
| 6:20-7:10 | 展示视觉匹配候选区域 |
| 7:10-8:00 | 生成报告，总结技术亮点 |

## 6. 文案口径

推荐表述：

> 本系统基于 SuperMap iDesktopX 进行任务区域数据整理和三维场景制作，基于 SuperMap iServer 发布三维、地图和空间数据服务，前端使用 SuperMap iClient3D for WebGL 展示低空任务场景，并集成航线规划、风险校验、动态重规划和视觉定位辅助能力。

边界表述：

> 本项目聚焦软件仿真验证和任务辅助规划，不接入真实无人机飞控，不生成真实飞行控制指令。

## 7. 验收标准

- 材料能够独立说明项目价值和技术路线。
- 演示视频不依赖临时口头解释也能看懂。
- PPT 和系统 UI 截图一致。
- 所有 SuperMap 使用点都有截图或功能体现。
