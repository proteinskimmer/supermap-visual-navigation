# 基于 SuperMap GIS 底座的低空视觉自主导航与三维仿真规划系统

## 1. 项目理解

本项目不是传统无人机飞控系统，也不以真实无人机接入和实飞控制为当前目标。项目本质是一个软件系统开发项目，核心在于基于 SuperMap GIS 底座，集成视觉识别、视觉定位、智能航线规划、动态重规划和三维仿真验证能力，形成一个可演示、可解释、可部署的低空任务智能规划平台。

项目可以面向应急救援、低空巡检、训练推演等场景。国防相关场景如需提及，应限定为仿真推演和训练验证，不涉及真实武器控制、打击任务或真实飞控执行。

一句话定位：

> 基于 SuperMap GIS 三维环境，集成 UAV 视觉影像地理重定位、后端融合导航状态、航线规划与风险重规划能力，实现低空任务的视觉自主导航软件仿真验证。

## 2. SuperMap 在项目中的角色

SuperMap 不负责直接实现视觉自主导航算法，也不自带完整的无人机智能飞行系统。它在本项目中的核心作用是提供 GIS 基础能力和三维空间环境。

### 2.1 SuperMap iDesktopX

用于数据生产和场景制作：

- 多源地理数据导入与整理。
- 遥感影像、DEM、矢量数据处理。
- 坐标系统一、裁剪、配准、制图。
- 工作空间和数据源制作。
- 三维场景制作。
- 为 iServer 发布服务准备数据。

### 2.2 SuperMap iServer

用于 GIS 服务发布：

- 地图服务发布。
- 三维服务发布。
- 数据服务发布。
- 空间分析服务发布。
- 为 Web 前端和 Python 后端提供 GIS REST 服务。

### 2.3 SuperMap iClient3D for Cesium

用于 Web 三维展示和交互：

- 加载 SuperMap 三维场景。
- 展示影像、地形、建筑、风险区、航线。
- 显示视觉匹配候选区域。
- 展示航线规划、动态重规划和风险校验结果。
- 实现三维仿真飞行、视角跟随、轨迹回放等效果。

### 2.4 SuperMap iObjects Python / Java

可选使用，用于更深度的后端 GIS 处理：

- 批量空间分析。
- 数据处理自动化。
- 离线数据包生成辅助。
- 航线规划前置 GIS 约束计算。

## 4. 核心功能规划

### 4.1 SuperMap 三维 GIS 环境构建

目标是先构建一个可用于视觉导航和航线规划的任务区域三维环境。

功能内容：

- 使用 iDesktopX 整理遥感影像、DEM、矢量数据。
- 制作任务区域三维场景。
- 使用 iServer 发布地图、三维、数据和空间分析服务。
- 前端使用 iClient3D 加载三维场景。

演示重点：

- 证明系统 GIS 能力不是普通底图，而是基于 SuperMap 数据生产、服务发布和三维展示实现。

### 4.2 视觉自主导航核心模块

目标是根据遥感影像/正射影像和 UAV 视角图像，估计当前图像对应的地理位置，并把视觉定位结果写入后端导航状态，驱动无人机融合位置、遥测、事件流和必要时的辅助/复核模式。

推荐流程：

1. 遥感影像预处理。
2. 将任务区域切分为候选瓦片。
3. 使用 DINOv2 做粗检索，找出相似候选区域。
4. 使用 LoFTR 或 LightGlue 做局部特征匹配。
5. 使用 RANSAC 做几何一致性验证。
6. 输出视觉位置、候选区域、置信度、匹配点数量、偏移估计。
7. 后端根据视觉结果生成 `fused_position`、`navigation_mode`、遥测和事件。
8. 前端从后端导航时间线读取状态，并在 SuperMap 三维场景中展示。

注意：

- 该模块不输出真实飞控控制指令。
- 该模块定位为软件仿真层面的视觉地理重定位和导航状态更新能力。
- 视觉辅助导航保留为低置信度、人工复核、GNSS 对照或演示兜底模式。

### 4.3 基于 GIS 约束的航线自动规划

目标是根据起点、目标点、任务区域和 GIS 约束自动生成候选航线。

输入数据：

- 起点、目标点、任务点。
- DEM 高程。
- 风险区、限制区、禁入区。
- 建筑物、道路、水体、植被等地物。
- 最低/最高高度约束。
- 航程、平滑度、安全距离等参数。

推荐算法：

- 主算法：A* 路径搜索。
- 可扩展：RRT*，用于连续空间或动态重规划展示。

规划模式：

- 最短航线。
- 最安全航线。
- 综合最优航线。

SuperMap 参与方式：

- 通过 iServer 或 iObjects 获取 GIS 数据。
- 使用 SuperMap 空间分析生成风险约束。
- 通过高程数据计算地形影响。
- 将规划结果在 SuperMap 三维场景中展示和校验。

### 4.4 动态避障与实时重规划仿真

目标是增强系统的智能性和演示效果。

仿真流程：

1. 系统生成初始推荐航线。
2. 仿真飞行开始。
3. 中途加入临时风险区，例如山火、塌方、临时禁飞区。
4. 系统检测当前航线与新风险区相交。
5. 触发局部重规划。
6. 原航线风险航段变红。
7. 新航线生成并接续到目标点。
8. 事件日志记录重规划原因和结果。

实现边界：

- 只做软件仿真。
- 不接真实飞控。
- 不生成真实无人机执行指令。

### 4.5 航线风险校验

目标是对人工航线和自动生成航线进行 GIS 风险分析。

校验内容：

- 是否穿越风险区。
- 是否超出任务边界。
- 是否接近障碍物。
- 是否低于安全高度。
- 是否存在高程突变。
- 是否经过不适宜区域。
- 是否超出最大航程。

输出内容：

- 风险等级。
- 风险航段高亮。
- 风险原因说明。
- 航线评分。
- 高程剖面。
- 可导出分析报告。

### 4.6 离线 GIS 数据包

目标是为断网或弱网环境下的仿真和任务分析提供离线数据支撑。

数据包内容：

- SuperMap 工作空间或数据源。
- 任务区域影像。
- DEM / 高程数据。
- 矢量图层。
- 三维场景配置。
- 航线和任务点数据。
- 风险区数据。
- 视觉匹配候选瓦片索引。
- 仿真配置。
- 数据说明和版本信息。

定位：

- 用于离线展示、仿真推演、任务辅助分析。
- 不作为真实飞控任务包。

### 4.7 三维仿真与可解释 UI

目标是把系统能力以清晰的任务故事演示出来。

界面结构：

- 中央：SuperMap 三维地图。
- 左侧：任务、图层、数据包。
- 右侧：视觉匹配、航线评分、风险告警。
- 底部：仿真时间轴和事件日志。
- 小窗：无人机视角画面或匹配结果。

仿真内容：

- 航迹动态推进。
- 视角跟随。
- 视觉匹配过程展示。
- 风险告警触发。
- 动态重规划过程展示。
- 任务事件记录。
- 任务报告导出。

## 7. 接真实无人机飞控与仿真的区别

当前项目建议以仿真验证为主，不接真实飞控。

### 7.1 仿真系统

特点：

- 航线是软件中的模拟路线。
- 位置是仿真位置。
- 告警是模拟或基于数据触发。
- 重规划结果只在地图和 UI 中展示。
- 无需处理真实通信、传感器误差和硬件安全。
- 更适合比赛开发和答辩演示。

### 7.2 接真实飞控

如果接真实无人机飞控，系统复杂度会显著提升：

- 需要接入 MAVLink 或厂家 SDK。
- 需要实时读取飞控状态。
- 需要处理 IMU、气压计、磁罗盘、视觉里程计等传感器数据。
- 需要将航线转换为飞控可执行任务格式。
- 需要处理通信延迟、中断、失控保护、低电量、返航等安全逻辑。
- 需要真实硬件测试和飞行安全审批。
- 算法错误可能影响真实设备行为。

因此，真实飞控对接不建议作为本次比赛范围。可以在文档中作为未来扩展方向。

## 11. 项目主线修订：从瓦片匹配演示升级为合成视图视觉自主导航

### 11.1 修订结论

本项目的核心主线正式调整为：

> 基于 DEM、正射影像和三维地理场景的合成视图匹配视觉自主导航仿真系统。

低空航线规划、风险校验、三维飞行播放和 SuperMap 场景展示不再被定义为项目最终成果本身，而是定义为验证视觉自主导航能力的仿真平台和任务环境。

项目真正要证明的是：无人机在 GNSS 不可靠、存在初始定位偏差或导航漂移时，能否利用机载视觉图像与预加载地理数据进行匹配，估计自身位置与偏差，并输出可解释的导航修正结果。

### 11.2 输入、输出和要解决的问题

项目输入包括：

- UAV 当前图像或仿真相机图像。
- 初始位姿、航线先验或任务区域范围。
- DEM/DSM 高程数据。
- 正射影像或遥感底图。
- 建筑物、道路、水体、风险区等 GIS 矢量数据。
- 相机内参、视场角、飞行高度、姿态等仿真参数。

项目输出包括：

- 视觉估计位置。
- 相对参考航线或先验位置的偏移量。
- 导航修正建议。
- 置信度、误差半径、匹配点数和失败原因。
- 视觉定位事件流。
- 供仿真平台验证的融合导航状态。

项目要解决的问题是：

> 在低空无人机场景中，当单纯依赖 GNSS 或预设航线不足以保证可靠定位时，如何利用预加载的 DEM、正射影像和三维 GIS 数据，对 UAV 视觉图像进行地图辅助重定位，并将定位结果转化为可解释、可验证的导航修正。

### 11.3 技术路线

推荐主链路如下：

```text
先验位置 / 任务航线 / 搜索范围
-> 候选区域粗检索
-> 基于 DEM + 正射影像 + 建筑/DSM 生成候选 UAV 合成视图
-> 真实或仿真 UAV 图像与合成视图匹配
-> RANSAC/优化估计位置、姿态或平面偏移
-> 输出视觉定位观测、误差半径、置信度和失败解释
-> 与仿真导航状态融合
-> 驱动 UAV 轨迹修正、重定位展示和任务报告
```

其中，当前已有的瓦片方案保留，但其定位降级为“候选区域粗检索”和“调试可视化索引”，不能再作为视觉自主导航的最终算法表述。

### 11.4 分阶段目标

v0.3：候选区域检索与预计算演示。

- 保留现有瓦片索引。
- 保留预计算匹配结果。
- 前端展示候选区域、置信度、偏移量和解释原因。
- 明确口径：这是视觉定位链路的粗检索/演示框架。

v0.4：DEM/正射影像合成视图原型。

- 根据候选 UAV 位姿、相机参数、DEM 地表和正射影像生成模拟 UAV 视图。
- 支持俯视或近俯视相机的合成图。
- 前端展示“当前 UAV 图像 vs 合成候选视图 vs 匹配结果”。
- 输出候选位姿、像素偏移、地面偏移和置信度。

v0.5：真实匹配算法接入。

- 接入 ORB/SIFT/LoFTR/LightGlue 等算法之一。
- 对合成视图与 UAV 图像进行特征匹配。
- 使用 RANSAC 或优化方法剔除错误匹配。
- 输出位置估计、误差半径、匹配点数、内点比例和失败原因。

v0.6：导航闭环仿真验证。

- 故意设置 UAV 初始位置偏差或导航漂移。
- 通过视觉定位输出修正量。
- 仿真平台展示 UAV 从偏移轨迹逐步回到参考航线或正确地理位置。
- 任务报告说明视觉定位对导航状态的贡献。

### 11.5 边界口径

- 本项目不输出真实飞控指令。
- 本项目不承诺真实无人机实时闭环飞行。
- 本项目输出的是软件仿真层面的视觉定位观测、导航修正建议和融合导航状态。
- 视觉辅助导航保留为降级模式：当匹配置信度不足、需要人工复核或外部定位可用时，视觉结果只作为参考提示，不直接修正导航状态。
- 答辩时应避免把“航线规划动画”说成核心成果，应强调“视觉定位如何驱动导航修正”。
